# 套件优化说明（未优化 → 优化）

本文档说明 IntelliTest **审查导出套件优化**（`scripts/optimize_export_cases.py`）
对综合流程导出结果（`未优化/`）做了哪些自动处理，产出 `优化/`。

## 输入与背景

- **原始需求**：`login_requirements.csv`（36 条 LR-001～LR-036）
- **目标应用**：OWASP Juice Shop（`target-apps/juice-shop`，默认 `http://localhost:3001`）
- **未优化导出**：`未优化/` — 本次 **138** 条（LLM 黑盒 + 白盒 ST 综合流程新导出）
- **优化中间结果**：`优化/` — **150** 条（脚本中间产物）
- **终稿用例**：`人工优化/` — **146** 条（本流水线交付目录）

## 套件优化做了什么

| 步骤 | 说明 | 本次数量 |
| --- | --- | ---: |
| 1. 剔除噪声 | 无关数值 BVA（-1/0/100）、Remember Me 无效 EP、与需求矛盾的 EP | 0 |
| 2. 去重 | 相同需求+技术+coverage+test_data 语义重复 | 0 |
| 3. 补全 test_data | 将 `valid_example` / 占位邮箱替换为 Juice Shop 种子账号 | 98 条修正 |
| 4. 修正 oracle | DT 条件为 False 时改为 401；客户端校验需求改为「按钮禁用」 | （含于修正） |
| 5. 规则补漏 | 脚本 `SUPPLEMENT_CASES` 补 XSS/SQLi/空格 BVA/JWT 等缺口 | 12 |

详细补漏清单见 `优化/覆盖率审查与变更说明.md` 与 `优化/变更清单.json`。

## 与上一轮（360 条规则引擎导出）的差异

本次未优化仅 **138** 条，因已启用 **LLM 黑盒**（EP/BVA/DT），不再批量生成
email/password 字段上的 `-1/0/100` 数值 BVA，故套件优化 **剔除数为 0**。
主要工作变为：**补全种子数据** + **12 条规则补漏** + **修正 DT oracle**。
